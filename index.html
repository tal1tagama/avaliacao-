Sobre o Projeto

Este projeto demonstra a utilização de uma esteira de automação CI/CD utilizando GitHub Actions. O objetivo é validar o código automaticamente e simular um processo de build para garantir maior confiabilidade no software.

Como funciona a Pipeline

1️⃣ Validação

O primeiro job verifica os arquivos do projeto e garante que o código esteja pronto para continuar no processo.

2️⃣ Build

O segundo job executa apenas se o primeiro passar e simula o build do projeto criando a pasta dist.

3️⃣ Secret

O pipeline utiliza um Secret do GitHub para demonstrar leitura segura de informações durante a execução.
